# DocShip
让我搜索一下还有哪些文档框架我们可能遗漏了：

没法搜索，让我根据我的知识来整理一下：

---

**我们已经覆盖的 (16个)：**
docsify, docusaurus, docute, docfx, eleventy, fumadocs, honkit, jekyll, mdbook, mkdocs-material, nextra, rspress, sphinx, starlight, vitepress, vuepress

---

**还可以考虑的（开箱即用，商业友好许可证）：**

| 框架 | 语言 | 许可证 | 特点 | 难度 |
|------|------|--------|------|------|
| **Docpress** | JS | MIT | 基于 Markdown，简单 | 低 |
| **Slate** | Ruby | Apache-2.0 | API 文档风格（三栏布局） | 中 |
| **Just the Docs** | Ruby | MIT | Jekyll 主题（我们已用） | ✅ 已有 |
| **Markdoc** | JS | MIT | Stripe 出品，需要配置 | 高 |
| **Lume** | Deno | MIT | Deno 生态的 SSG | 中 |
| **Pagic** | Deno | MIT | Deno + React | 中 |
| **Docus** | JS | MIT | Nuxt 3 文档主题 | 中 |
| **Nuxt Content** | JS | MIT | Nuxt 生态 | 中 |
| **Saber** | JS | MIT | Vue 生态（不太活跃） | 低 |
| **Gridsome** | JS | MIT | Vue 生态（不太活跃） | 中 |

---

**值得做的推荐（按优先级）：**

1. **Docus** (Nuxt 3) - MIT，现代化，Vue 生态但我们还没有 Nuxt 的
2. **Slate** - Apache-2.0，API 文档三栏布局，很多公司用
3. **Lume** - MIT，Deno 生态，轻量快速

---

**不推荐的：**
- Saber、Gridsome - 项目不太活跃了
- Markdoc - 需要太多配置
- Pagic - 太小众

你对哪个感兴趣？**Docus** 或 **Slate** 我觉得比较值得做。
